# Deployment

## Clean installation

To completely recreate the project from scratch:

```bash
docker compose down -v --remove-orphans
docker compose -f docker-compose.yml --env-file .env_local up -d --build
```

This procedure:

* removes all project containers;
* removes all project volumes (including the PostgreSQL database);
* recreates the database;
* applies all Prisma migrations;
* executes the database seed;
* creates the initial protected administrator account;
* starts all application services.

The application should be fully operational after a single `docker compose up --build` without requiring any manual database setup.

---

## Administrator account

The initial administrator account is created automatically by the Prisma seed using the values defined in the environment file:

```text
ADMIN_EMAIL
ADMIN_PASSWORD
```

The initial administrator is marked as a protected administrator:

* cannot be deleted;
* cannot be demoted;
* is intended to guarantee permanent administrative access to the application.
